# 검토서 템플릿 가이드 (report_template.docx)

검토서(.docx)의 **글꼴·색·표 스타일·페이지 설정·머리말/꼬리말**은 모두 이 폴더의
`report_template.docx` 한 파일로 제어됩니다. 스타일을 바꾸려면 **코드 수정 없이 이 파일만
Word로 편집**하면 됩니다. 앱은 실행 시 이 템플릿을 불러와 서식을 그대로 상속합니다.

---

## 동작 방식

```
build_review_docx()
  ├─ assets/report_template.docx 있음  → 이 템플릿을 열고 본문만 비운 뒤 콘텐츠 생성
  │                                       (스타일·페이지설정·머리말/꼬리말은 그대로 상속)
  └─ 없음                               → 기본 문서 + 코드 내장 기본 글꼴로 폴백
```

- 템플릿의 **본문 내용은 무시**됩니다(코드가 지우고 다시 채움). 표지 같은 고정 페이지가
  필요하면 **머리말/꼬리말**에 넣으세요(이 부분은 유지됩니다).
- 제목·표·본문은 아래 **스타일 이름**으로 생성되므로, 그 스타일을 편집하면 결과가 바뀝니다.

---

## 코드가 사용하는 스타일 이름

| 문서 요소 | 스타일 이름 | 편집 위치 (Word) |
|---|---|---|
| 문서 표제 "활주로 방향 검토서" | `Title` (제목) | 홈 → 스타일 → '제목' 우클릭 → 수정 |
| 절 제목 (1. 검토 개요 …) | `Heading 1` (제목 1) | 스타일 → '제목 1' 수정 |
| 하위 제목 (4.1 풍배도 …) | `Heading 2` (제목 2) | 스타일 → '제목 2' 수정 |
| 본문 문단·캡션 | `Normal` (표준) | 스타일 → '표준' 수정 |
| 모든 표 | `Table Grid` (표 격자) | 아래 '표 스타일' 참고 |

> **목차(TOC)는 `제목 1`·`제목 2` 스타일을 기준**으로 자동 생성됩니다. 제목 스타일만
> 잘 유지하면 목차가 정확히 잡힙니다.

---

## 스타일 편집 방법 (Word)

1. `assets/report_template.docx`를 Word로 엽니다.
2. **글꼴/색/크기**: 홈 탭 → 스타일 그룹에서 대상 스타일(예: '제목 1') 우클릭 → **수정**.
   - 한글 글꼴이 깨지지 않게 하려면 수정 창에서 글꼴을 **한글 지원 글꼴**(맑은 고딕, 바탕 등)로
     지정하세요. (기본 템플릿은 맑은 고딕)
3. **페이지 여백/크기**: 레이아웃 탭 → 여백/크기.
4. **머리말/꼬리말·페이지 번호**: 삽입 → 머리글/바닥글. (기본 템플릿은 바닥글 가운데 페이지번호)
5. **저장**: 반드시 같은 이름 `report_template.docx`로, **Word 문서(.docx)** 형식 저장.

---

## 표 스타일 바꾸기

두 가지 방법 중 하나:

- **간단**: Word에서 '표 격자(Table Grid)' 스타일 자체를 원하는 디자인으로 수정.
  코드가 이 이름을 쓰므로 자동 반영됩니다.
- **커스텀**: Word에서 '새 표 스타일'을 만들어 이름을 지정한 뒤(예: `검토서표`),
  `app.py`의 `REPORT_TABLE_STYLE = "Table Grid"` 를 그 이름으로 바꿉니다.
  (해당 스타일이 템플릿에 없으면 자동으로 'Table Grid'로 폴백)

---

## 목차 갱신

문서를 처음 열면 목차 자리에 안내 문구가 보입니다. **목차 위에서 F9**
(또는 우클릭 → **필드 업데이트**)를 누르면 제목과 페이지번호가 채워집니다.
HWP에서도 해당 필드를 업데이트하면 목차가 생성됩니다.

---

## 적용 흐름 (배포 반영)

```
report_template.docx 편집(Word)  →  git add assets/report_template.docx
   →  git commit  →  git push  →  Streamlit Cloud 자동 재배포  →  새 서식 적용
```

로컬에서 바로 확인하려면 앱을 재실행한 뒤 분석 → 검토서 다운로드로 결과를 보면 됩니다.

---

## 주의

- 템플릿 파일이 없거나 손상되면 앱은 **기본 서식으로 폴백**하여 계속 동작합니다(오류 아님).
- 표지 '페이지'는 코드가 본문을 비우므로 유지되지 않습니다. 고정 브랜딩은
  머리말/꼬리말을 쓰거나, 필요하면 표지 자동 생성 기능을 별도로 요청하세요.
